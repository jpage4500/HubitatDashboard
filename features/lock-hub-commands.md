# Lock Hub Commands

By default, device tiles are unprotected - meaning anyone can click on one to send a command to the Hub; for example, locking the front door or opening the garage.

You can require the user to enter a PIN any time a command is sent to the hub or just for a specific device(s)

![](<../.gitbook/assets/image (176).png>)

Enabling this option will prompt you for a 4-digit PIN. Enter it and re-enter to confirm.

Once a PIN is set, trying to send a command to the hub will require a PIN

![](<../.gitbook/assets/image (175).png>)
